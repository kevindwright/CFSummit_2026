# Why move to queryExecute()?

- Code organization and maintainability
- Parameterization clarity
- Parameters, options, and the SQL string are all in one function call

It treats database access as a function call instead of a special language construct, resulting in code that is more testable, reusable, maintainable.

# Aligned with modern software engineering practices

- `<cfquery>` was designed for page-centric ColdFusion development. 
- queryExecute() was designed for application-centric ColdFusion development.
