# Why use queryExecute?


- Parameters, options, and the SQL string are all in one function call

- iterates over every element of an array and accumulates a single result value, which can be any type — a number, string, struct, array, etc
- Think of it as "folding" the whole array down into one value, one element at a time.

# No more loops! 🎉

- Benefits are cleaner, more declarative functional programming code
- Drawback - processing overhead of invoking a closure (or lambda function) on every single iteration
